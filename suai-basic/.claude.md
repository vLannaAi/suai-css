# SUAI CSS Framework - AI Development Guidelines

You are working with the SUAI (Semantic, Utility-first, Adaptive Interface) CSS Framework. This document defines mandatory rules for maintaining SUAI standards and ensuring consistency across the entire project.

---

## 🚨 CRITICAL RULES - MUST FOLLOW ALWAYS

### Rule 1: NO SCOPED STYLES IN VUE COMPONENTS - EVER

**This is the most important rule. Violating this breaks the entire architecture.**

```vue
<!-- ❌ WRONG - DO NOT DO THIS -->
<template>
  <article class="card">
    <h2>Title</h2>
  </article>
</template>

<style scoped>
.card { color: red; }
.card h2 { font-size: 2rem; }
</style>
```

```vue
<!-- ✅ CORRECT - ALWAYS DO THIS -->
<template>
  <article class="card">
    <h2>Title</h2>
  </article>
</template>

<style scoped>
/* All card component styles are in assets/styles/modules/card.css */
</style>
```

**Why:** Scoped styles break SMACSS organization, prevent component reuse, and make styles unmanageable. They make it impossible to maintain consistent theming across the application.

---

### Rule 2: SMACSS Layer Organization is Mandatory

Every CSS file MUST follow this structure:

```css
/* {Layer} - Description of what this file styles
   Features: X.X.X Feature Name, X.X.X Feature Name, ... */

@layer {layer_name} {
  /* Feature X.X.X: Description of component/element */
  .component-name {
    /* styles here */
  }
}
```

**Valid layers:**
- `@layer base` - HTML element defaults (h1-h6, p, a, lists)
- `@layer layout` - Page structure and containers
- `@layer components` - Reusable UI components
- `@layer state` - Dynamic states and theme variations
- `@layer theme` - Theme-specific styling

**Example:**
```css
/* Modules - Card component for displaying content
   Features: 1.1.2 Single Root Class, 2.1.3 Dark Mode, 4.1.3 Component Variants */

@layer components {
  .card {
    @apply p-6 rounded-xl bg-white;
    @apply dark:bg-gray-900;
  }
}
```

---

### Rule 3: Single Root Class Pattern is Required

**ONE component = ONE class. All children styled via CSS selectors, not classes.**

```vue
<!-- ❌ WRONG - Multiple classes on children -->
<article class="card">
  <header class="card-header">
    <h3 class="card-title">Title</h3>
  </header>
  <section class="card-body">
    <p class="card-text">Content</p>
  </section>
</article>
```

```vue
<!-- ✅ CORRECT - Single root class only -->
<article class="card">
  <header>
    <h3>Title</h3>
  </header>
  <section>
    <p>Content</p>
  </section>
</article>
```

**CSS should use descendant selectors:**
```css
.card {
  /* Root styling */
  @apply p-6 rounded-xl bg-white;

  /* Child styling via descendant selectors */
  > header {
    @apply flex items-center justify-between;
    @apply mb-4 pb-4 border-b;
  }

  > header > h3 {
    @apply text-xl font-bold;
  }

  > section > p {
    @apply text-gray-600;
  }
}
```

**Benefits:**
- Clean HTML without class clutter
- Styles are self-documenting
- Easier to refactor
- Better for AI code generation

---

### Rule 4: Multi-line Utility Organization (3-Group Pattern)

Always organize `@apply` directives into **exactly 3 groups** with blank lines:

1. **Layout & Flow** - flex, grid, spacing, sizing, positioning
2. **Appearance** - colors, borders, shadows, typography, radius
3. **Dark Mode & States** - dark:, hover:, active:, focus:, transitions

```css
.component {
  /* Group 1: Layout & Flow */
  @apply flex items-center justify-between;
  @apply w-full h-auto;
  @apply p-6 md:p-8;
  @apply gap-4;

  /* Group 2: Appearance */
  @apply bg-white rounded-xl;
  @apply border-2 border-gray-200;
  @apply shadow-md;
  @apply text-gray-900;

  /* Group 3: Dark Mode & States */
  @apply dark:bg-gray-900 dark:border-gray-700;
  @apply hover:shadow-lg;
  @apply transition-all duration-300;
}
```

**Why:** This makes utilities predictable and easy to scan for any AI reading the code.

---

### Rule 5: Use Semantic HTML Always

Prefer semantic elements over generic `<div>` tags:

```html
<!-- ❌ WRONG - Div soup -->
<div class="article-container">
  <div class="article-header">
    <h2>Article Title</h2>
  </div>
  <div class="article-content">
    <p>Content here</p>
  </div>
  <div class="article-footer">
    <button>Share</button>
  </div>
</div>
```

```html
<!-- ✅ CORRECT - Semantic elements -->
<article class="article">
  <header>
    <h2>Article Title</h2>
  </header>
  <section>
    <p>Content here</p>
  </section>
  <footer>
    <button>Share</button>
  </footer>
</article>
```

**Semantic elements to use:**
- `<header>` - Top section of component
- `<footer>` - Bottom section of component
- `<article>` - Independent content (post, card, comment)
- `<section>` - Grouped content
- `<main>` - Primary page content
- `<nav>` - Navigation
- `<aside>` - Supplementary content (sidebar, callout)
- `<figure>` - Image with caption
- `<figcaption>` - Caption for figure

---

### Rule 6: Prefix System for Classes

Use specific prefixes to indicate component type and state:

| Prefix | Purpose | Example |
|--------|---------|---------|
| None | Component name | `.card`, `.button`, `.modal` |
| `.l-` | Page-level layout | `.l-grid`, `.l-sidebar` |
| `.is-` | Dynamic state | `.is-active`, `.is-loading`, `.is-disabled` |
| `.theme-` | Theme variant | `.theme-dark`, `.theme-compact` |
| `--` | Component modifier | `.card--featured`, `.button--large` |

**Examples:**
```html
<!-- Components (no prefix) -->
<div class="card"></div>
<div class="modal"></div>

<!-- States (is- prefix) -->
<div class="card is-active"></div>
<button class="button is-loading"></button>

<!-- Layouts (l- prefix) -->
<div class="l-grid"></div>

<!-- Themes (theme- prefix) -->
<div class="card theme-dark"></div>

<!-- Modifiers (-- suffix) -->
<div class="button--large"></div>
```

---

### Rule 7: Advanced Selectors Over Extra Classes

Use CSS selectors to reduce HTML classes:

```css
/* Direct child combinator */
.card > header h2 { }

/* Adjacent sibling combinator */
.element + .element { margin-top: 0; }

/* General sibling combinator */
.element ~ .element { }

/* Pseudo-elements for decorative content */
.card::before { content: ""; }
.card::after { content: ""; }

/* :not() for exclusion */
.item:not(:last-child) { margin-bottom: 1rem; }

/* Attribute selectors */
[data-state="active"] { }
[data-color="primary"] { }
```

**Benefits:**
- Less HTML clutter
- More maintainable CSS
- Clearer intent
- Self-documenting code

## 🔧 Complete Workflow for New Components

### Step 1: Create Vue Component

Create component in `pages/` or `components/` with semantic HTML and single root class:

```vue
<template>
  <section class="feature-showcase">
    <h2>Features</h2>
    <div>
      <article>
        <h3>Feature 1</h3>
        <p>Description</p>
      </article>
    </div>
  </section>
</template>

<style scoped>
/* All feature-showcase styles are in assets/styles/modules/feature-showcase.css */
</style>
```

### Step 2: Create CSS File

Create file: `assets/styles/modules/feature-showcase.css`

```css
/* Modules - Feature showcase component
   Features: 1.1.2 Single Root Class, 1.2.2 Advanced Selectors, 2.1.2 Responsive, 2.1.3 Dark Mode */

@layer components {
  /* Feature 1.1.2: Single root class - Main component */
  .feature-showcase {
    /* Layout & Flow */
    @apply max-w-6xl mx-auto;
    @apply px-6 py-12 md:py-16;

    /* Appearance */
    @apply bg-white rounded-lg;
    @apply border border-gray-200;

    /* Dark Mode & States */
    @apply dark:bg-gray-900 dark:border-gray-700;
    @apply transition-all duration-300;

    /* Feature 1.2.2: Direct child selectors for structure */
    > h2 {
      /* Layout & Flow */
      @apply mb-8 pb-4;

      /* Appearance */
      @apply text-3xl font-bold;
      @apply text-primary;

      /* Dark Mode */
      @apply dark:text-primary-light;
      @apply border-b-2 border-primary;
    }

    > div {
      /* Layout & Flow */
      @apply grid grid-cols-1 md:grid-cols-3;
      @apply gap-6;

      > article {
        /* Layout & Flow */
        @apply p-4;

        /* Appearance */
        @apply bg-gray-50 rounded-lg;

        /* Dark Mode */
        @apply dark:bg-gray-800;

        > h3 {
          /* Appearance */
          @apply text-lg font-semibold;
          @apply mb-2;
        }

        > p {
          /* Appearance */
          @apply text-sm text-gray-600;

          /* Dark Mode */
          @apply dark:text-gray-400;
        }
      }
    }
  }

  /* Feature: State variant for featured */
  .feature-showcase.is-featured {
    /* Appearance */
    @apply ring-2 ring-primary;
    @apply shadow-lg;
  }
}
```

### Step 3: Import CSS

Add import to `assets/styles/main.css`:

```css
/* Modules - Reusable component styles */
@import 'modules/feature-showcase.css';
```

### Step 4: Test

Verify:
- [ ] Component renders correctly
- [ ] Dark mode works (check `dark:` variants)
- [ ] Responsive design works (resize browser)
- [ ] Hover/interactive states work
- [ ] No console errors

- [ ] **No missed imports** - CSS file imported in `main.css`
- [ ] **File in right location** - CSS in correct SMACSS directory

## ❌ Anti-Patterns to AVOID

**Never do any of these:**

1. ❌ **Scoped styles** in Vue components with actual CSS
   ```vue
   <style scoped>
   .my-class { color: red; }  <!-- WRONG! -->
   </style>
   ```

2. ❌ **Multiple classes** on child elements
   ```html
   <article class="card">
     <header class="card-header">  <!-- WRONG! -->
     <h2 class="card-title">       <!-- WRONG! -->
   </article>
   ```

3. ❌ **Inline styles** in HTML
   ```html
   <div style="color: red; font-size: 18px;">  <!-- WRONG! -->
   ```

4. ❌ **Single-line utilities** without grouping
   ```css
   .component { @apply flex items-center bg-white dark:bg-gray-900 p-6 rounded-lg border border-gray-200 dark:border-gray-700 shadow-md; }  <!-- WRONG! -->
   ```

5. ❌ **Missing @layer** directives
   ```css
   .component { @apply p-6; }  <!-- WRONG - no @layer! -->
   ```

6. ❌ **No feature documentation** in CSS files
   ```css
   .component { @apply p-6; }  <!-- WRONG - missing feature comment! -->
   ```

7. ❌ **Div soup** instead of semantic HTML
   ```html
   <div class="card">
     <div class="card-header">    <!-- WRONG - use <header>! -->
       <div class="card-title">   <!-- WRONG - use <h3>! -->
   ```

8. ❌ **Classes without prefixes**
   ```html
   <div class="active"></div>           <!-- WRONG - should be .is-active -->
   <div class="sidebar"></div>          <!-- WRONG - should be .l-sidebar -->
   <div class="dark-theme"></div>       <!-- WRONG - should be .theme-dark -->
   ```

9. ❌ **CSS file not imported** in main.css
   ```css
   /* If you create modules/my-component.css, MUST add to main.css */
   @import 'modules/my-component.css';
   ```

10. ❌ **Missing dark mode** support
    ```css
    .component { @apply bg-white; }  <!-- WRONG - no dark:variant! -->
    /* CORRECT: */
    .component { @apply bg-white dark:bg-gray-900; }
    ```

## 🎯 Decision Tree: When to Add a Class?

**Q: Should I add a class to this element?**

1. Is it a **new component root**? → YES, add single root class (e.g., `.card`)
2. Is it a **dynamic state**? → YES, use `.is-*` prefix (e.g., `.is-active`)
3. Is it a **page-level layout**? → YES, use `.l-*` prefix (e.g., `.l-sidebar`)
4. Is it a **theme variant**? → YES, use `.theme-*` prefix (e.g., `.theme-dark`)
5. Can you style it with **CSS selectors** in parent? → NO, don't add class
6. Otherwise → Don't add a class, use CSS selector instead

**Examples:**

```html
<!-- Root component - add class -->
<article class="testimonial">  <!-- ✅ Need class for root -->
  <header>                      <!-- ❌ No class - style via .testimonial > header -->
    <p>Quote</p>               <!-- ❌ No class - style via .testimonial > header > p -->
  </header>
  <footer>                      <!-- ❌ No class -->
    <cite>Author</cite>         <!-- ❌ No class -->
  </footer>
</article>

<!-- State - use .is- prefix -->
<article class="testimonial is-featured">  <!-- ✅ .is-featured for state -->

<!-- Layout - use .l- prefix -->
<div class="l-grid">           <!-- ✅ .l-grid for layout -->

<!-- Theme - use .theme- prefix -->
<div class="card theme-dark">  <!-- ✅ .theme-dark for theme -->
```

## 🚀 Maintenance & Updates

### When something breaks:

1. Check if CSS file is imported in `main.css`
2. Check if CSS is in `<style scoped>` (WRONG - should be external)
3. Check for conflicting scoped styles from other components
4. Check dark mode - verify `dark:` variants exist
5. Check responsive - verify breakpoint variants exist

### When adding features:

1. Create in correct SMACSS directory
2. Add feature documentation header
3. Use `@layer` directive
4. Follow single root class pattern
5. Import in `main.css`

### When refactoring:

1. Never move CSS into Vue scoped styles
2. Always maintain external CSS structure
3. Keep feature documentation updated
4. Test dark mode and responsive
5. Verify all imports in main.css

## 🎯 Summary

The SUAI framework is built on **three core principles:**

1. **Semantic**: Use meaningful HTML elements
2. **Utility-first**: Organize CSS with Tailwind/UnoCSS utilities
3. **Adaptive**: Support dark mode, responsive, and interactive states

**To follow SUAI correctly:**

```
Semantic HTML + External CSS + SMACSS @layers + Single Root Class + 3-Group Utilities = SUAI Compliance
```

**Remember:**
- ✅ All CSS goes in external files
- ✅ All CSS wrapped in `@layer`
- ✅ One root class per component
- ✅ Children styled via CSS selectors
- ✅ Utilities organized in 3 groups
- ✅ Semantic HTML always
- ✅ Dark mode support required
- ✅ Responsive variants required

**This ensures your code is consistent, maintainable, and AI-friendly.**
