 Structure: <header> + <nav> + <main> + <footer> | Mobile: Drawer | Desktop: Layered grid (nav full-height behind header/footer)
View CSS Selectors (No Classes!) 
/* Mobile: Hidden drawer */
form > div[data-layout="side-nav"] > nav {
  position: fixed;
  left: 0;
  transform: translateX(-100%);
  transition: transform 0.3s;
}

/* Tablet+: Layered grid layout */
@media (min-width: 768px) {
  form > div[data-layout="side-nav"] {
    display: grid;
    grid-template-columns: 280px 1fr;
  }

  /* Nav extends full height, layered behind header/footer */
  form > div[data-layout="side-nav"] > nav {
    position: static;
    transform: none;
    grid-row: 1 / -1; /* Full height */
    z-index: 10; /* Behind header/footer */
  }

  /* Footer spans full width over nav */
  form > div[data-layout="side-nav"] > footer {
    grid-column: 1 / -1; /* Full width */
    z-index: 20; /* Above nav */
  }
}
Element Semantic Meaning When to Use
<nav> Navigation links section Major navigation blocks, site/app navigation
data-* attributes Custom data attributes Store state for CSS selectors (data-nav-open="true")

Structure: <header> + <main> + <footer> (sticky) | Mobile: Bottom bar | Desktop: Top toolbar
View CSS Selectors 
/* Mobile: Sticky bottom */
form > div[data-layout="bottom-toolbar"] > footer {
  position: sticky;
  bottom: 0;
  display: flex;
  justify-content: space-around;
}

/* Desktop: Top toolbar */
@media (min-width: 768px) {
  form > div[data-layout="bottom-toolbar"] > footer {
    position: static;
    order: -1; /* Move to top */
  }
}

Mobile-First Responsive Strategy

All layouts follow mobile-first principles with progressive enhancement through media queries.

Breakpoint Strategy Layout Changes
Mobile (<768px) Base styles, single column Stacked layout, drawer nav, bottom toolbar
Tablet (≥768px) Side-by-side layouts Nav sidebar, aside visible, grid layouts
Desktop (≥1024px) Multi-column, wider spaces Article grids, wider aside, more spacing

Key CSS Techniques Used

Technique Purpose Example Selector
Child combinator (>) Direct children only form > div > header
Attribute selectors Target by data attributes div[data-layout="side-nav"]
Descendant combinator Any nested element main article header
Pseudo-classes State-based styling :hover, :last-child, :first-child
CSS Grid 2D layouts grid-template-columns: 280px 1fr
Flexbox 1D layouts, alignment display: flex; flex-direction: column
Media queries Responsive breakpoints @media (min-width: 768px)

SUAI Layout Elements Summary

This page demonstrates responsive, mobile-first layouts using pure semantic HTML with zero classes. All styling uses CSS selectors, combinators, and attribute selectors.

Key Principles

  • Mobile-First: Base styles for mobile, progressive enhancement for larger screens
  • Semantic HTML: Right element for right purpose (nav, article, aside, section)
  • Zero Classes: All styling through element selectors and combinators
  • CSS Grid & Flexbox: Modern layout techniques for responsive design
  • Progressive Enhancement: Works everywhere, better on modern browsers
