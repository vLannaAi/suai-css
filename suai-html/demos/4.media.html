Accessibility Best Practices

Media elements must be accessible to all users, including those using assistive technologies.

Element Accessibility Requirement Implementation
<img> Alternative text Always include descriptive alt attribute
<video> Captions for deaf/HoH Provide <track kind="captions">
<video> Audio descriptions for blind Provide <track kind="descriptions">
<audio> Transcript for deaf Provide text transcript near audio element
<figure> Clear caption Use <figcaption> to explain the content
All media Keyboard controls Native controls attribute provides keyboard access
All media No autoplay with sound Avoid autoplay or use with muted

SUAI Media Elements Summary

This page demonstrates all semantic media elements in the SUAI framework using the PLA theme. All elements use zero classes and pure semantic HTML.

Key Principles

  • Semantic HTML: Use the right element for the right purpose
  • Accessibility First: Always include alt text, captions, and descriptions
  • Progressive Enhancement: Provide multiple formats and fallbacks
  • Responsive Media: Use <picture> for art direction
  • Native Controls: Browser controls provide keyboard accessibility

Element Coverage

Media Elements:
├── <img>          - Embedded images
├── <picture>      - Responsive images with multiple sources
├── <source>       - Media source specification
├── <figure>       - Self-contained content grouping
├── <figcaption>   - Caption for figures
├── <audio>        - Audio content
├── <video>        - Video content
└── <track>        - Text tracks for media (captions, subtitles)
