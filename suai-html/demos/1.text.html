Text - Inline

Inline text elements that modify content within paragraphs. SUAI provides strong semantic differentiation through visual styling.

Visual Demo

In the world of web development, semantics matter. When we emphasize content, we're not just making it look different—we're adding meaning to our markup.

Consider this HTML Living Standard quote: User agents must not consider this element to be a heading. The specification uses HTML to structure documents, created by Tim Berners-Lee in 1991.

Foreign words like résumé use the <i> tag. Mathematical formulas like H2O or E=mc2 use subscript and superscript. Highlighted text draws attention without semantic weight.

Fine print: This text is smaller for legal disclaimers or side comments. Outdated information appears struck through. Proper names in Chinese use underline to indicate non-pronunciation-based writing.

Text - Block

Block-level text elements that structure content and define document hierarchy.

Visual Demo

Primary Heading (h1)

This is the main heading, typically used once per page for the page title or main topic.

Section Heading (h2)

Secondary headings divide content into major sections.

Subsection Heading (h3)

Tertiary headings break down sections into subsections.

Minor Heading (h4)

Fourth-level headings for detailed breakdown.

Small Heading (h5)

Fifth-level headings for fine-grained structure.

Smallest Heading (h6)

The smallest heading level, rarely needed in most documents.

Paragraphs and Breaks

Paragraphs contain the main body text. Each paragraph should focus on a single idea or topic.

Multiple paragraphs create visual rhythm and improve readability through proper spacing.
